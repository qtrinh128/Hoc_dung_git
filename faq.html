Staging Area

Trong GIT Staging area là một khu vực trung gian lưu trữ những thay đổi của chúng ta trên tệp tin (chỉnh sửa tệp tin) để có thể commit

Muốn commit thì bắt buộc tệp tin đó phải nằm trong Staging area

Để đưa 1 tệp tin vào Staging area thì ta dùng lệnh: git add ten_file hoặc git add * * là toàn bộ nội các tệp tin có sự thay đổi và đưa chúng vào Staging area

Một tệp tin nằm trong Staging sẽ có trạng thái là Stagged

Commmit

Hiểu đơn giản thì Commit nó sẽ lưu lại 1 bản snapshot những sự thay đổi trong tệp tin, thư mục. Lưu ý các tệp tin và thư mục này phải nằm trong Staging area

Mỗi lần lần commit nó sẽ lưu lại lịch sử chỉnh sửa, kèm theo tên và địa chỉ email của người commit

Để thực hiện commit ta sử dụng lệnh: git commit -m "message"

Sau khi commit thì ta có thể push lên repository bằng lệnh: git push origin master

Điều kiện để có thể commit 1 tệp tin

Cần đưa tệp tin đó vào trạng thái tracked bằng lệnh: git add *

Lưu ý:Tệp tin trong trạng thái tracked cũng sẽ có các trạng thái phụ khác

  • Unmodified: Tệp tin chưa chỉnh sửa gì
  • Modified: Tệp tin đã được chỉnh sửa, ở trạng thái này ta sẽ không thể commit chúng, muốn commit trước tiên ta phải đưa nó về trạng thái tracked bằng lệnh: git add *
  • Tracked: Tệp tin sẵn sàng đề commit

Lưu ý: ta có thể đưa một tệp tin đã được tracked để commit mà không cần đưa nó vào Staging area với tham số lênh -a trong lệnh: git commit -a -m "message"
